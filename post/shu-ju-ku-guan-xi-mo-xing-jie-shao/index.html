数据库——关系模型介绍

关系数据库的结构

单一的数据结构----关系

现实世界的实体以及实体间的各种联系均用关系来表示

数据的逻辑结构----二维表

从用户角度，关系模型中数据的逻辑结构是一张二维表

数据库模式

属性：A1,A2,…,AnA_1, A_2, …, A_nA1​,A2​,…,An​
关系模式：R=(A1,A2,…,An)R = (A_1, A_2, …, A_n)R=(A1​,A2​,…,An​)

Example:

instructor=(ID,name,dept_name,salary)\mathrm{instructor} = (ID, name, dept\_name, salary)instructor=(ID,name,dept_name,salary)

形式上，给定集合D1，D2，....。DnD_1，D_2，....。D_nD1​，D2​，....。Dn​一个关系r是D1D1D1 x D2D2D2 x ... x DnD_nDn​的子集。

因此，一个关系是一组n−mathrmtuples。(a1,a2,...,an)n-mathrm{tuples}。(a_1, a_2, ..., a_n)n−mathrmtuples。(a1​,a2​,...,an​) 其中每一个ai都在Dia_i都在D_iai​都在Di​中。

关系模式表现了型的概念，它代表着表的框架；关系表现为值的概念,成为关系实例(relation instance\mathtt{relation\ instance}relation instance)

一个关系模式下可以建立多个关系，例如在学生关系的关系模式下，可以为全校所有班级各建一个学生表

表是动态的，是数据库中数据的快照

关系数据库是关系的集合，其中每个关系都有自己的关系模式

元组在关系中出现的顺序是无关紧要的(元组可以按任意顺序存储)

Example: 有无序元组的教员关系。

关系模式：Instructor(ID，name，dept_name,salary)\mathrm{Instructor}(ID，name，dept\_name,salary)Instructor(ID，name，dept_name,salary)

码

  1. 超码(superkey)

    一个或多个属性的集合，这些属性的集合可以使我们在一个关系中唯一地标识一个元组。

  2. 候选码（Candidate Key）

    关系中的一个属性组，其值能唯一标识一个元组。若从属性组中去掉任何一个属性，它就不具有这一性质了，这样的属性组称作候选码。

    • 任何一个候选码中的属性称作主属性。

    • 最小的超码。

  3. 主码（Primary Key）

    进行数据库设计时，从一个关系的多个候选码中选定一个作为主码，如可选定 D#D\#D# 作为 DEPTDEPTDEPT 的主码

  4. 外码（Foreign Key）

    关系 RRR 中的一个属性组，它不是 RRR 的码，但它与另一个关系S的码相对应，则称这个属性组为R的外部码，如 SSS 关系中的 D#D\#D# 属性

    RRR 是参照关系(Referencing Relation)

    SSS 是被参照关系(Referenced Relation)

    RRR 通过外码 FFF 参照关系 SSS

  5. 参照完整性约束（referential integrity constraint）

    在参照关系中任意元组在特定属性上的取值必然等于被参照关系中某个元组在特定属性上的取值

模式图

  1. 含有主码和外码依赖的数据库模式可以用模式图（schema diagram）来表示

    关系：矩形

    关系名称：列在矩形上方

    属性：列在矩形内

    主码属性：下划线标注

    外码依赖：从参照关系的外码属性到被参照关系的主码属性之间的箭头表示

关系查询语言

  1. 查询语言：用户用来从数据库中请求获取信息的语言

  2. 分类：过程化语言（procedural language）、非过程化语言（nonprocedural language）

    过程化语言：用户指导系统对数据库执行一系列操作以计算出所需结果

    非过程化语言：用户描述所需信息，不用给出获取该信息的具体过程

  3. 实际使用的查询语言既包含过程化方法，又包含非过程化方法，例如SQL（Chapter 3~Chapter 5）

  4. “纯”查询语言（Chapter 6）

    过程化：关系代数

    非过程化：关系演算、域关系演算

关系运算

自然连接：两个关系上的自然连接运算所匹配的元组在两个关系共有的所有属性上取值相同

让r和s分别是模式RRR和SSS上的关系。

那么，关系RRR和SSS的 "自然连接 "是在模式R/cupSR/cup SR/cupS上得到如下的关系。

  • 考虑来自r的每一对元组tr和来自s的每一对元组ts。

  • 如果 tr 和 ts 在 R∪SR \cup SR∪S 的每个属性上都有相同的值，则在结果中添加一个 tuple，其中

    • t在r上的值与trt_rtr​相同。

    • t在s上的值与tst_sts​相同。
