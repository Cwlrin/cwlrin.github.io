SQL查询语言概览

  • SQL最早版本是 IBM 开发的 Sequel，20 世纪 70 年代早期作为 System R 项目的一部分
  • Sequel语言发展至今，名称已变为SQL （Structured Query Language，结构化查询语言）
  • ANSI（美国国家标准化组织） and ISO（国际标准化组织） 颁布了 SQL标准:
    • SQL-86, SQL-89, SQL-92
    • SQL:1999, SQL:2003, SQL:2008

SQL语言的组成

  1. 数据定义语言（Data-Definition Language，DDL）：SQL DDL 提供定义关系模式、删除关系以及修改关系模式的命令
  2. 数据操纵语言（ Data-Manipulation Language，DML ）：SQL DML 提供从数据库中查询信息，以及在数据库中插入元组、删除元组、修改元组的能力
  3. 完整性（integrity）：SQL DDL包括定义完整性约束的命令，保存在数据库中的数据必须满足所定义的完整性约束。破坏完整性约束的更新是不允许的
  4. 视图定义（view definition）：SQL DDL 包括定义视图的命令
  5. 事务控制（transaction control）：SQL 包括定义事务的开始和结束的命令
  6. 嵌入式SQL和动态SQL（embedded SQL and dynamic SQL）：嵌入式和动态 SQL 定义 SQL 语句如何嵌入到通用编程语言，如 C、C++、Java
  7. 授权（authorization）：SQL DDL 包括定义对关系和视图的访问权限的命令

SQL数据定义

  • char(n) ：固定长度的字符串，用户可指定长度 nnn。
  • varchar(n). ：可变长度的字符串，用户可指定最大长度 nnn。
  • int. : 整数（整数的有限子集，与机器有关）。
  • smallint : 小整数（整数域类型的机器依赖性子集）。
  • numeric(p,d) : 定点数字，用户指定精度为 ppp 位，小数点右边有 nnn 位
  • real, double precision. : 浮点和双精度浮点数，精度随机器而定。
  • float(n). 浮点数，用户指定的精度至少为 nnn 位。

alter table

  1. alter table r add A D
    • A 为要加入到关系 rrr 中的一个属性，D 为属性 A 的域
    • 关系中的所有元组在新属性上的取值将被设置为null

    • 许多数据库并不支持去掉属性

SQL查询的基本结构

SQL DML语言 支持查询、插入、删除、更新元组的操作

单关系查询

  1. 通过select 子句能够得到目标属性上的查询结果

    对应于关系代数中的投影映射

    例如: find the names of all instructors（找出所有教师名字）

    select name from instructor

NOTE: SQL 的名称是不分大小写的

E.g. Name ≡ NAME ≡ name

有的人在我们使用粗体字的地方都是用大写。

SQL 的查询结果中可能出现重复.

select dept_name from instructor

在 select 后使用关键词 distinct 可以强制消除重复查询结果.

Eg：查询所有教师所在的系名，并去掉重复记录

select distinct dept_name
from instructor

关键词 all 可以显式指明不去掉重复,

select all dept_name
from instructor

select 子句中的 ∗*∗ 代表“所有属性”

select *
from instructor

select 子句还可以包含 +,–,∗,/+, –, *, /+,–,∗,/ 运算符的算术表达式，运算对象可以是常数或元组的属性

select ID, name, salary*1.1
from instructor

返回一个与 instructor 一样的关系, 只是属性 salary 的值是原来的1.1倍.

这并不导致对 instructor 关系的任何改变

where子句

where 子句允许只选出那些在 from 子句的结果关系中满足特定谓词的元组

Eg：找出所有在Comp. Sci. 系且 salary > 80000 的教师姓名

select name from instructor
where dept_name = 'Comp. Sci.' and salary > 80000

SQL允许在 where子句中使用逻辑连接词 and, or, not.

逻辑连接词可以是包含比较运算符 >,>=,<,<=,=,<>> , >= , < ,<= ,= , <>>,>=,<,<=,=,<> 的表达式

where 子句中的谓词可以限制笛卡尔积所建立的组合，只留下那些对所需答案有意义的组合

查询讲授课程的教师，不输出没讲授课程的教师

select name，course_id
from instructor，teaches
where instructor.ID=teaches.ID

找出 Comp.Sci 系的教师名和课程标识

select name，course_id
from instructor，teaches
where instructor.ID=teaches.ID  and 
instructor.dept_name=‘Comp.Sci’;

SQL查询的含义可理解为：

  1. 为 from 子句中列出的关系产生笛卡尔积
  2. 在步骤 1 的结果上应用 where 子句中指定的谓词
  3. 对步骤 2 结果中的每个元组，输出 select 子句中指定的属性

多关系查询

eg：找出所有教师的姓名，以及他们所在系的名称和系所在建筑的名称

  • instructor：dept_name 系名
  • department：building 建筑名称

需要访问的关系列在 from 子句中，并且在 where 子句中指定匹配条件

select name，instructor. dept_name ，building
from instructor，department
where instructor. dept_name = department . dept_name

关系名可被用作前缀来说明使用的是哪个属性

当需要把同一个关系的两个不同元组信息组合时，可以使用更名运算

SQL查询的基本结构:

select A1, A2, ..., An
from r1, r2, ..., rm
where P

AiA_iAi​代表一个属性

RiR_iRi​代表一种关系

PPP 是一个谓词

SQL 查询的结果也是关系.

以运算顺序理解 select 子句：from，where，select

from 子句定义了一个在该子句中所列出关系的笛卡尔积，可用集合理论来形式化定义

for each 元组 t1 in 关系 r1
	for each 元组 t2 in 关系 r2
		……

    	把t1，t2，……，tm连接成单个元组t

此结果关系具有来自 from 子句中所有关系的所有属性

eg：关系instructor和teaches的笛卡尔积关系模式：

自然连接

自然连接作用于两个关系，并产生一个关系作为结果。仅考虑在两个关系模式中都出现的属性上取值相同的元组对

select * from instructor natural join teaches;

共同属性：course_id

查询：对于大学中所有授课的教师，找到他们的姓名以及所讲述的所有课程

select name, course_id
from instructor, teaches
where instructor.ID = teaches.ID;

select name, course_id
from instructor natural join teaches;

自然连接需要注意: 两个关系中代表不同含义的属性具有相同的名称，按照该属性产生错误的自然连接
对于大学中所有授课的教师，找到他们的姓名以及所讲述的所有课程
错误的版本(使course.dept_name = instructor.dept_name)

select name, title
from instructor natural join teaches natural join course;

正确的版本

select name, title
from instructor natural join teaches, course
where teaches.course_id = course.course_id;

另一个正确的版本

select name, title
from (instructor natural join teaches)
join course using(course_id);

字符串运算

集合运算

在有重复的关系中，SQL可以定义结果中出现多少个元组的副本。

一些关系代数运算符的多集版本--给定多集关系 r1r_1r1​ 和 r2r_2r2​。

  1. σθ\sigma_\thetaσθ​(r1r_1r1​): 如果 r1r_1r1​ 中的元组 t1t_1t1​ 有 c1c_1c1​ 个副本，且 t1t_1t1​ 满足选择 ，那么 σθ\sigma_\thetaσθ​(r1r_1r1​) 中就有t1t_1t1​ 的 c1c_1c1​ 个副本。
  2. ∏A\prod_A∏A​（rrr）中。对于 r1r_1r1​ 中每一个元组 t1t_1t1​ 的副本，∏A\prod_A∏A​(r1r_1r1​)中都有一个元组 ∏A\prod_A∏A​(t1t_1t1​)的副本，其中 ∏A\prod_A∏A​(t1t_1t1​)表示单元组 t1t_1t1​ 的投影。
  3. r1×r2r_1×r_2r1​×r2​：如果 r1r_1r1​ 中的元组 t1t_1t1​ 有 c1c_1c1​ 个副本， r2r_2r2​中的元组 t2t_2t2​ 有 c2c_2c2​ 个副本，则元组 t1t_1t1​ 有 c1×c2c_1×c_2c1​×c2​ 个副本

SQL作用在关系上的union、intersect、except运算对应于集合论中的 ∪,∩,−∪,∩,-∪,∩,− 运算

并运算

2009年秋季学期 和 2010年春季学期 开设所有课程集合

(select course_id from section where sem = ‘Fall’ and year = 2009)
union
(select course_id from section where sem = ‘Spring’ and year = 2010)

intersect
(select course_id from section where sem = ‘Spring’ and year = 2010)

(select course_id from section where sem = ‘Spring’ and year = 2010)

  • 全部自动去掉重复
  • union all, intersect all ， except all保留所有重复
  • 假设一个课程在关系 rrr 中出现 mmm 次，在关系 sss 中出现 nnn 次

min(m,n) times in r intersect all s
max(0, m – n) times in r except all s

空值

元组的某些属性可以有一个空值，用null表示

null 代表值未知或者值不存在.

如果算术表达式的任一输入为 null 则该表达式的结果为 null

Example: 5 + null returns null

谓词 is null 可以用来测试一个属性值是否为空

Example: 查找所有薪水为 null 的讲师。

select name
from instructor
where salary is null

SQL 将涉及空值的任何比较运算的结果视为 unknown

Example: 5 < null or null <> null or null = null

where 子句的谓词中可以对比较结果使用 and、or、not 的布尔运算，这些布尔运算的定义也被扩展到可以处理unknown值
OR:

  1. (unknown or true) = true,
  2. (unknown or false) = unknown
  3. (unknown or unknown) = unknown
    AND:
  4. (true and unknown) = unknown,
  5. (false and unknown) = false,
  6. (unknown and unknown) = unknown
    NOT:
  7. (not unknown) = unknown

“P is unknown” ：如果 P 为 unknown ，值为true

如果 where 子句谓词对一个元组计算结果为 false 或 unknown，则该元组不能加入到结果集中

聚集函数

聚集函数是以值的一个集合（集或多重集）为输入，返回单个值的函数

  • avg: average value
  • min: minimum value
  • max: maximum value
  • sum: sum of values
  • count: number of values

sum 和 avg 的输入必须是数字集

其他运算符可作用在非数字数据类型的集合上，如字符串

基本聚集

查询 Computer Science 系教师的平均工资(保留重复元组)

select avg (salary) (as avg_salary)
from instructor
where dept_name= ’Comp. Sci.’;

查询2010年春季学期讲授一门课程的教师总数（多个课程段仅算做一门课）

select count (distinct ID)
from teaches
where semester = ’Spring’ and year = 2010

查询关系 course 中的所有元组数

select count (*)
from course;

SQL不允许count(*)和distinct同时使用

分组聚集

查询每个系的平均工资

select dept_name, avg (salary)
from instructor
group by dept_name;

Note: 没有教师的院系不会出现在查询结果中

出现在 select 语句中单没有被聚集的属性只能是出现在 group by 子句中的属性，即任何没有出现在 group by子句中的属性如果出现在select子句中，它只能出现在聚集函数内部，否则查询结果出错

/* erroneous query */
select dept_name, ID, avg (salary)
from instructor
group by dept_name;

having子句

查询所有院系中平均工资超过42000的院系名称和平均工资

查询条件并不针对单个元组，而是针对 group by 子句构成的分组

having 子句中的谓词在形成分组后才起作用

select dept_name, avg (salary)
from instructor
group by dept_name
having avg (salary) > 42000;

Note : 任何出现在having子句中，但没有被聚集的属性必须出现在 group by 子句中

包含聚集、group by、having子句的查询的含义可通过以下操作序列定义：

  • 与不带聚集的查询类似，最先根据 from 子句来计算出一个关系
  • 如果出现了 where 子句，where 子句中的谓词将应用到 from 子句的结果关系上
  • 如果出现了 group by 子句，满足 where 谓词的元组通过 group by 子句形成分组。如果没有 group by 子句，满足 where 谓词的整个元组集被当成一个分组
  • 如果出现了 having 子句，它将应用到每个分组上；不满足 having 子句谓词的分组将抛弃
  • select 子句利用剩下的分组产生出查询结果中的元组，即在每个分组上应用聚集函数来得到单个结果元组

对空值和布尔值的聚集

求工资总和（Total all salaries）

select sum (salary )	from instructor

  • sum 运算符忽略输入中的 null 值

  • 如果不存在非空输入值，则返回 null

除 count(*) 外所有的聚集函数都忽略输入集合中的空值

当参加函数运算的输入值集合为空集：

  • count 运算值为 0
  • 其他所有运算返回 null

嵌套子查询

SQL提供嵌套子查询机制

子查询：子查询是嵌套在另一个查询中的 select-from-where 表达式

子查询嵌套在where子句中，通常用于对集合的成员资格、集合的比较以及集合的基数进行检查

集合成员资格

查询Fall 2009 and in Spring 2010同时开课的所有课程

select distinct course_id
from section
where semester = ’Fall’ and year= 2009 and 
	course_id in (select course_id
		from section
		where semester = ’Spring’ and year= 2010);

查询Fall 2009开课， Spring 2010不开课的课程

select distinct course_id
from section
where semester = ’Fall’ and year= 2009 and
	course_id  not in (select course_id
    	from section
        where semester = ’Spring’ and year= 2010);

in 和 not in 操作符也可用于枚举集合

select distinct name
from instructor
where name not in (‘Mozart’ , ‘Einstein’);

查询（不同的）学生数，他们选修了ID 10101的教师所讲授的课程段

select count (distinct ID)
from takes
where (course_id, sec_id, semester, year) in
	select course_id, sec_id, semester, year
    from teaches
    where teaches.ID= 10101);

集合的比较

eg：查询工资至少比 Biology 系某一个教师工资高的所有教师的姓名

select distinct T.name
from instructor as T, instructor as S
where T.salary > S.salary and S.dept_name = ’Biology’;

“至少大于某一个”在SQL中可以用 > some 子句表示

select name
from instructor
where salary > some (select salary
    from instructor
    where dept_name = ’Biology’);

eg：查询工资比 Biology 系每个教师工资都高的所有教师姓名.

select name
from instructor
where salary > all (select salary
    from instructor
    where dept_name = ’Biology’);

类似some，SQL允许 <all <=all >=all =all <>all

eg：查询2009年秋季学期和2010年春季学期同时开课的所有课程

select course_id
from section as S
where semester = ’Fall’ and year= 2009 and
	exists (select * from section as T

相关子查询（correlated subquery）：来自外层查询的一个相关名称可以用在where子句的子查询中，使用了来自外层查询相关名词的子查询被称为相关子查询

包含了子查询的查询中，在相关名称上可以应用作用域规则

作用域规则：在一个子查询中只能使用此子查询本身定义的，或者在包含此子查询的任何查询中唯一的相关名词。如果一个相关名称既在子查询中定义，又在包含该子查询的查询中定义，则子查询中的定义有效

eg：选修了 Biology 系开设的所有课程的学生.

select distinct S.ID, S.name
from student as S
where not exists ( (select course_id
    from course	
    where dept_name = ’Biology’)
    except
    (select T.course_id
    from takes as T
    where S.ID = T.ID));

重复元组存在性测试

unique：测试在一个子查询的结果中是否存在重复元组

如果作为参数的子查询结果中没有重复的元组，unique 结构返回 true 值

eg：查询在2009年最多开设一次的课程

select T.course_id
from course as T
where unique (select R.course_id
	from section as R
	where T.course_id= R.course_id
	and R.year = 2009);
select T.course_id
from course as T
where 1>= (select count (R.course_id)
	from section as R
	where T.course_id= R.course_id
	and R.year = 2009);

from子句中的子查询

SQL 允许 from 子句中使用子查询表达式
eg：查询系平均工资超过$42,000的那些系中教师的平均工资.

select dept_name, avg_salary
from (select dept_name, avg (salary) as avg_salary
	from instructor
	group by dept_name)
where avg_salary > 42000;

方法2：as子句为子查询结果关系起名，并对属性进行重命名

select dept_name, avg_salary
from (select dept_name, avg (salary)
	from instructor
	group by dept_name)
	as dept_avg (dept_name,  avg_salary)
where avg_salary > 42000;

from子句嵌套的子查询中不能使用来自from子句其他关系的相关变量

SQL：2003 lateral 子句作为前缀，访问from子句中在它前面的表或子查询中的属性

select name, salary, avg_salary
from instructor I1, 
	lateral (select avg(salary) as avg_salary
	from instructor I2
	where I2.dept_name= I1.dept_name);

Note: 只有少数SQL实现支持 lateral 子句，如IBM DB2

with子句

with 子句提供定义临时关系的方法，这个定义只对包含with子句的查询有效.

eg：查询具有最大预算值的系

with max_budget (value) a
	(select max(budget)
	from department)
select budget
from department, max_budget
where department.budget = max_budget.value;

with子句实在SQL：1999 中引入的，目前有许多数据库系统都提供了支持

With 子句使查询在逻辑上更加清晰，允许在一个查询内的多个地方使用视图的定义

eg：查询所有工资总额大于所有系平均工资总额的系

with dept _total (dept_name, value) as
        (select dept_name, sum(salary)
         from instructor
         group by dept_name),
dept_total_avg(value) as
       (select avg(value)
       from dept_total)
select dept_name
from dept_total, dept_total_avg
where dept_total.value >= dept_total_avg.value;

标量子查询_select嵌套

标量子查询（Scalar subquery）：SQL允许子查询出现在返回单个值的表达式能够出现的任何地方，只要该子查询只返回包含单个属性的单个元组，这样的子查询称为标量子查询
Eg：查询所有的系以及他们拥有的教师数

select dept_name,
	(select count(*) from instructor 
	where department.dept_name = instructor.dept_name)
	as num_instructors
from department;

更新 update

eg：年度教师工资增长，收入高于$100,000 的教师工资增长 3%, 其余的教师工资增长 5%

写两个更新语句：

update instructor
	set salary = salary * 1.03
	where salary > 100000;
update instructor
	set salary = salary * 1.05
	where salary <= 100000;

注意update语句的顺序

case语句：在一条update语句中实现多种更新

update instructor
	set salary = case
		when salary <= 100000 then salary * 1.05
		else salary * 1.03
		end

case语句的一般格式：

case
    when pred1 then result1
    when pred2 then result2
        ......
    when predn then resultn
    else result0
end

eg：把每个student元组的tot_cred属性值设为该生成功完成的课程学分总和

update student S
	set tot_cred = ( select sum(credits) from takes natural join course
		where S.ID= takes.ID and 
			takes.grade <> ’F’ and
			takes.grade is not null);

将没有上过任何课程的学生的total_creds设置为null。

不使用sum(credits)，而使用：

case
	when sum(credits) is not null then sum(credits)
	else 0
end
