数据库——高级SQL

使用程序设计语言访问数据库

API (application-program interface) 应用程序接口，连接数据库服务器

应用可以：

  • 连接数据库
  • 发送 SQL 命令给数据库服务器
  • 把结果存入程序变量中，每次一个元组。

通用编程语言的两种方法访问SQL：

  • 动态SQL：
    1. JDBC (Java Database Connectivity) works with Java
    2. ODBC (Open Database Connectivity) works with C, C++, C#, and Visual Basic. Other API’s such as ADO.NET sit on top of ODBC
  • 嵌入式SQL

JDBC

JDBC是Java语言的应用程序接口。定义了java程序连接数据库服务器的应用程序接口。

JDBC提供了多种查询和更新数据的特性，并取回查询。

JDBC 还支持元数据检索，例如，查询关系在哪个数据库中，属性名和类型

与数据库通信的模型:

  1. 打开一个数据库连接Open a connection
  2. 创建一个“语句”对象Create a “statement” object
  3. 使用语句对象发送语句执行语句并返回结果Execute queries using the Statement object to send queries and fetch results
  4. 异常机制处理错误Exception mechanism to handle errors

ODBC

开放数据库互连（Open DataBase Connectivity (ODBC) ）标准

  • 为连接数据库服务器写的标 准应用程序。
  • pplication program interface (API) to
    1. 打开数据库连接open a connection with a database,
    2. 发送查询和更新send queries and updates,
    3. 取回结果get back results.

应用程序，如，GUI, 电子表格，等，可以使用ODBC

嵌入式 SQL

SQL标准定义了嵌入SQL到许多不同的语言中，如C, C++, Java, Fortran, 和PL/1,

SQL查询所嵌入的语言被称为宿主语言（host language）, 宿主语言中使和SQL结构被称为嵌入式SQL（embedded SQL）.

这些语言的基本形式沿用了系统 R 将 SQL 嵌入 PL/1 的形式。

EXEC SQL 语句用来使预处理器识别嵌入式SQL请求。

EXEC SQL <embedded SQL statement >;

Note: 语法依赖宿主语言。

  • 例如，COBOL, 语句中的分号用 END-EXEC 代替
  • 在Java中嵌入使用 # SQL { .... };

在执行任意SQL语句前，程序必须首先连接数据库，使用：

EXEC-SQL connect to  server  user user-name using password;

这里，server 标识将要建立连接的服务器

在嵌入的SQL语句中可以使用宿主语言的变量，不过前面要加上冒号(😃 以区别于SQL变量。，(e.g., :credit_amount )

如此使用的变量必须声明在一个DECLARE区段里，见下面的代码。但声明变量的语法还是因循宿主语言的惯例。

EXEC-SQL BEGIN DECLARE SECTION
int  credit-amount ;
EXEC-SQL END DECLARE SECTION;

写一个嵌入式SQL语句，我们要用下面语句

declare c cursor for  <SQL query>

表达式中，变量 c 被称为该查询的游标

Example:

假设我们有一个宿主变量 credit_amount ，找出学分高于 credit_amount 的所有学生的名字。

SQL查询语句如下

EXEC SQL
declare c cursor for
select ID, name
from student
where tot_cred > :credit_amount
END_EXEC

变量 c 用来标识该查询。

open statement :

EXEC SQL open c ;

这条语句使得数据库系统执行这条查询并把执行结果存于一个临时关系中。当open语句被执行的时候，宿主变量credit-amount的值就会被应用到查询中。

Fetch语句要求结果关系的每一个属性有一个宿主变量相对应

EXEC SQL fetch c into :si, :sn END_EXEC

一条单一fetch请求只能得到一个元组。如果我们想得到所有的结果，程序中必须包含对所有元组执行的一个循环。

close 语句来告诉数据库系统删除用于保存查询结果的临时关系。

EXEC SQL close c ;

Note: 细节依赖于特定语言的细节。

例如，Java嵌入定义了Java迭代器来步入结果元组。

对数据库修改（update, insert, and delete）嵌入式SQL表达式

通过游标来更新数据库关系。如，要为音乐系每个老师的salary属性都增加100，可以声明这样的游标：

EXEC SQL 
declare c cursor for
select *
from instructor
where dept_name = ‘Music’
for update

然后我们利用在游标上的fetch操作对元组进行迭代（就像我们先前看到的例子一样），每取到一个元组我们都执行以下的代码。

update instructor
set salary = salary + 1000
where current of c

函数和过程

SQL:1999支持函数和过程

  • SQL允许定义函数、过程和方法。定义可以通过SQL的有关过程的组件，也可以外部的程序设计语言，例如C, Java
  • 用外部语言编写的函数对于特殊的数据类型（如图像和几何对象）特别有用。
  • 例如：检查多边形是否重叠，或比较图像是否相似的函数。

一些数据库系统支持表值函数，它可以返回一个关系作为结果。

SQL:1999还支持一组丰富的命令式构造，包括循环，if-then-else，赋值。

许多数据库对SQL有专有的程序化扩展，与SQL:1999不同。

SQL 函数Functions

定义一个函数，给定一个系的名子，返回该系的教师数目。

create function dept_count (dept_name varchar(20))
returns integer
begin
declare d_count  integer;
    select count (* ) into d_count
    from instructor
    where instructor.dept_name = dept_name
return d_count;
end

这个函 数dept_count 可以用在返回教师数大于12的所有系的名称和预算的查询中。

select dept_name, budget
from department
where dept_count (dept_name ) > 12

复合语句: begin … end

在begin 和 end之间会包含复杂的SQL语句.

returns -- 显示返回的变量类型，如，integer

return -- 作为调用函数的结果，返回指定的值

SQL 函数可以被看作带参数的视图（parameterized views ），它通过允许参数把视图的概念更加一般化。

表函数Table Functions

SQL:2003标准支持返回关系作为结果的函数（表函数）

Example: 返回一个包含某特定系的所有教师的表（图5-6）

create function instructor_of (dept_name char(20))
	returns table  (  
        ID varchar(5),
        name varchar(20),
        dept_name varchar(20),
        salary numeric(8,2))
	return table
        (select ID, name, dept_name, salary
        from instructor
        where instructor.dept_name = instructor_of.dept_name)

该函数的使用：查询返回“MUSIC”系的所有教师

select *
from table (instructor_of (‘Music’))

SQL过程 Procedures

dept_count 函数也可以写成一个过程：

create procedure dept_count_proc (in dept_name varchar(20), out d_count integer)
begin
	select count(*) into d_count
    from instructor
    where instructor.dept_name = dept_count_proc.dept_name
end

可以从一个SQL过程中或者从嵌入式SQL中使和call 语句调用过程。

declare d_count integer;
call dept_count_proc( ‘Physics’, d_count);

过程和函数可以通过动态SQL触发。

SQL:1999允许多个函数或过程同名(called name overloading), ，只要同名过程的参数个数不同。或至少有一个参数类型不同。

支持过程和函数的语言构造 Language Constructs for Procedures & Functions

SQL所支持的构造赋予了它与通用程序设计语言相当的几乎所有的功能。
Warning: most database systems implement their own variant of the standard syntax below.
复合语句 begin … end,
在begin和end之间会包含复杂的SQL语句。
可以在复合语句中声明局部变量。
While 和 repeat 语句:

while 布尔表达式 do
    语句序列 ;
end while

repeat
    语句序列;
until  布尔表达式
end repeat

For循环：允许对查询的所有结果重复执行
Example: 查询所有系的预算

declare n integer default 0;
for r  as
    select budget from department
do
    set n = n + r.budget
end for

条件语句 (if-then-else)

SQL:1999 也支持 case 语句，类似于C语言中的 case语句

Example procedure: 定义过程，首先确认选课学生数没有超过该果所在教室的容量，然后完成学生对该课的注册。

返回 0 表示成功，返回 -1 表示超过教室容量

SQL支持发信号通知异常条件（exception conditions）,以及声明句柄（handlers）来处理异常

declare out_of_classroom_seats  condition
declare exit handler for out_of_classroom_seats
begin
...
..
signal out_of_classroom_seats
end

这个句柄是出口，一但条件发生，将会采取动作终止begin…and 中的语句。

其他动作是后续可能异常语句。

SQL:1999 允许用一种程序设计语言定义函数，如C或C++

外部过程和函数可以这样指定：

create procedure dept_count_proc(in dept_name varchar(20), out count integer)
language C
external name ’ /usr/avi/bin/dept_count_proc’
create function dept_count(dept_name varchar(20))
returns integer
language C
external name ‘/usr/avi/bin/dept_count’

Benefits of external language functions/procedures:

  • 对很多运算更高效，表达更准确。

Drawbacks

  • 执行函数的代码可能需要加载到数据库系统并执行。
    • 程序中的错误可能破坏数据库内部的结构
    • 安全风险，允许用户访问未授权数据
  • 关心安全性的数据库系统一般会将这些代码作为一个单独进程 一部分来执行，通过进程间通信，传入参数的值，取回结果
  • 如果数据库系统关心执行的效率胜过安全性则可以采用这种方式执行过程。

触发器Triggers

触发器（ trigger ）是一条语句，当对数据库作修改时，它自动被系统执行。

要设置触发器机制，必须满足两个要求：

  • 指明什么条件下执行触发器
  • 指明触发器执行时的动作

尽管触发器在SQL:1999之前并不是SQL标准的一部分，但是它仍被广泛地应用在基于SQL的数据库中。

触发器事件和动作 in SQL

触发器事件可以是insert, delete或 update

对于更新来说，触发器可以指定哪个属性使其执行

  • 如， after update of takes on grade 指定当更新关系takes的属性grade时执行触发器

Referencing语句建立变量，用来在插入完成后存储所插入行的值。

  • referencing old row as :建立一个变量用来存储已经更新或删除行的旧值。
  • referencing new row as : 用于插入和更新操作
    触发器可以在事件之前激发。在事件之前被执行的触发器可以作为避免非法更新、插入或删除的额外约束。如，假设所插入分数的值为空白则表明该分数发生缺失，将这个值用NULL值代替。Set语句可以用来执行这样的修改。
create trigger setnull_trigger before update of takes
referencing new row as nrow
for each row
when (nrow.grade = ‘ ‘)
begin atomic
    set nrow.grade = null;
end;

级触发器语句

我们可以对引起插入、删除或更新的SQL语句执行单一动作，而不是对每个被影响的行执行一个动作。

  • 使用 for each statement 子句来替代 for each row
  • 使用 referencing old table 或 referencing new table 来指向包含所有的被影响的行的临时表（称为过渡表transition tables)
  • 过渡表不能用于before触发器，无论它们是语句触发器还是行触发器，无论它们是语句触发器还是行触发器。
  • 在过渡表的基础上，一个单独的SQL语句就可以用来执行多个动作。

何时不用触发器

触发器早期被用于如下情况：

  • 维护统计数据（如，每个院系的总工资）
  • 复制或者备份数据库，在每一个关系的插入、删除或更新的操作上创建触发器，将改变记录在称为change或delta的关体系上。一个单独的进程将这些改变复制到数据库的副本。

现在，有更好的方法完成这些

  • 许多数据库现在支持物化视图，由数据库系统自动维护汇总数据。
  • 现代的数据库系统提供内置的数据库复制工具。

在许多情况下，可以使用封装工具代替触发器

  • 定义方法来更新域
  • 作为更新方法的一部分执行动作，来代替触发器

触发器动作的非故意执行

  • 当数据从一个备份的拷贝中加载
  • 一个站点上的数据库更新复制到备份站点时
  • 在这些动作执行之前，触发器执行可以无效

触发器的其他风险

  • 运行期间一个触发器错误会导致引发该触发器的动作语句失败。
  • 无限的触发链（Cascading execution）
