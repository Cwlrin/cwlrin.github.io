# BCNF（巴克斯范式）

定义：关系模式 R∈1NFR∈1NFR∈1NF，若 X→YX→YX→Y 且 Y⊈XY⊈XY⊈X 时，XXX 必含有码，则关系模式 R∈BCNFR∈BCNFR∈BCNF。

也就是说，当 3NF3NF3NF 消除了主属性对码的部分函数依赖和传递函数依赖，则称为 BCNFBCNFBCNF。

结论：一个满足 BCNFBCNFBCNF 的关系模式，应有如下性质：

  1. 所有非主属性对每一个码都是完全函数依赖；

  2. 所有主属性对每一个不包含它的码，也是完全函数依赖；

  3. 没有任何属性完全函数依赖于非码的任何一组属性。

例：R(Pno,Pname,Mname)R(Pno,Pname,Mname)R(Pno,Pname,Mname) 的属性表示零件号、零件名和厂商名，如果约定，每种零件号只有一个零 件名，但不同的零件号可以有相同的零件名；每种零件可以有多个厂商生产，但每家厂商生产的零 件应有不同的零件名。函数依赖集如下：

(Pname,Mname)→Pno,Pno→Pname(Pname,Mname)→Pno, Pno→Pname(Pname,Mname)→Pno,Pno→Pname 候选码为(Pname,Mname)(Pname,Mname)(Pname,Mname) 或 (Pno,Mname)(Pno,Mname)(Pno,Mname)

分解为：R1(Pno,Pname)∈BCNFR1(Pno,Pname)∈BCNFR1(Pno,Pname)∈BCNF

R2(Pno,Mname)∈BCNFR2(Pno,Mname)∈BCNFR2(Pno,Mname)∈BCNF

# 4NF（第四范式）

定义：关系模式 R∈1NFR∈1NFR∈1NF，若对于 RRR 的每个非平凡多值依赖 X→→YX→→YX→→Y 且 Y⊈XY⊈XY⊈X 时，XXX 必含有码，则关系模式 R(U,F)∈4NFR(U,F)∈4NFR(U,F)∈4NF。

4NF4NF4NF 是限制关系模式的属性间不允许有非平凡且非函数依赖的多值依赖。

注意：如果只考虑函数依赖，关系模式最高的规范化程度是 BCNFBCNFBCNF，如果考虑多值依赖，关系模式最高的规范化程度是 4NF4NF4NF。

# Armstrong 公理系统

Armstrong 公理系统（函数依赖的公理系统）：设关系模式 R(U,F)R(U,F)R(U,F)，其中 UUU 为属性集，FFF 是 UUU 上的一 组函数依赖，那么有如下推理规则：

  1. A1A1A1 自反律：若 Y⊆X⊆UY⊆X⊆UY⊆X⊆U，则 X→YX→YX→Y 为 FFF 所蕴涵。
  2. A2A2A2 增广律：若 X→YX→YX→Y 为 FFF 所蕴涵，且 Z⊆UZ⊆UZ⊆U，则 XZ→YZXZ→YZXZ→YZ 为 FFF 所蕴涵。
  3. A3A3A3 传递律：若 X→Y,Y→ZX→Y,Y→ZX→Y,Y→Z 为 FFF 所蕴涵，则 X→ZX→ZX→Z 为 FFF 所蕴涵。

根据上述三条推理规则又可推出下述三条推理规则：

  1. 合并规则：若 X→Y,X→ZX→Y,X→ZX→Y,X→Z，则 X→YZX→YZX→YZ 为 FFF 所蕴涵。
  2. 伪传递律：若 X→Y,WY→ZX→Y,WY→ZX→Y,WY→Z，则 XW→ZXW→ZXW→Z 为 FFF 所蕴涵。
  3. 分解规则：若 X→Y,Z⊆YX→Y,Z⊆YX→Y,Z⊆Y，则 X→ZX→ZX→Z 为 FFF 所蕴涵。

# 函数依赖的闭包 F+F^+F+

定义：关系模式 R(U,F)R(U,F)R(U,F) 中为 FFF 所逻辑蕴含的函数依赖的全体称为 F 的闭包，记为：F+F^+F+。

例：R(U,F)，U=(A,B,C,D),F={A→B,B→C,AC→D}R(U,F)，U=(A,B,C,D),F=\{A→B,B→C,AC→D\}R(U,F)，U=(A,B,C,D),F={A→B,B→C,AC→D}

# 属性的闭包 XF+X_F^+XF+​

定义：设 FFF 为属性集 U 上的一组函数依赖，X⊆UX⊆UX⊆U，XF+={A∣X→A能由F根据Armstrong公理导出}X_F^ +=\{A|X→A能由F根据Armstrong 公理导出\}XF+​={A∣X→A能由F根据Armstrong公理导出}，则称 XF+X_F^ +XF+​ 为属性集 X 关于函数依赖集 F 的闭包。

即：属性集 XXX 的闭包 XF+X_F^ +XF+​ 是指所有能由 XXX​ 决定的属性集合。

# 候选码的求解方法

给定一个关系模式 R(U,F),U={A1,A2,…,An}R(U,F),U=\{A_1,A_2,…,A_n\}R(U,F),U={A1​,A2​,…,An​}，FFF 是 RRR 的函数依赖集，那么，可以将属性分为如下四类：

  • LLL：仅出现在函数依赖集 F 左部的属性
  • RRR：仅出现在函数依赖集 F 右部的属性
  • LRLRLR：在函数依赖集 F 左右部都出现的属性
  • NLRNLRNLR：在函数依赖集 F 左右部都未出现的属性

根据候选码的特性，对于给定一个关系模式 R(U,F)R(U,F)R(U,F)，可以得出如下结论：

  • 结论 1：若 X(X⊆U)X(X⊆U)X(X⊆U) 是 L 类属性，则 XXX 必为 RRR 的任一候选码的成员。若 XF+=UX_F^ +=UXF+​=U，则 XXX 必为 RRR 的唯一候选码。
  • 结论 2：若 X(X⊆U)X(X⊆U)X(X⊆U) 是 R 类属性，则 XXX 不是 RRR 的任一候选码的成员。
  • 结论 3：是 X(X⊆U)X(X⊆U)X(X⊆U) 是 NLR 类属性，则 XXX 必为 RRR 的任一候选码的成员
  • 结论 4：若 X(X⊆U)X(X⊆U)X(X⊆U) 是 L 类和 NLR 类属性组成的属性集，若 XF+=UX_F^+=UXF+​=U，则 XXX 必为 RRR 的唯一候选码

第 1 步，根据题意，将所有的属性分类：

  • LLL：只在左边出现，一定是
  • RRR：只在右边出现，一定不是
  • LRLRLR：左右都出现，有可能是，也有可能不是
  • NLRNLRNLR：左右都没出现，一定是

第 2 步，将所有的 LLL 类和 NLRNLRNLR 类属性组合起来，设为 PPP ，求其闭包 PF+P_F^+PF+​，如果是全集 UUU，那么它就是候选码。

第 3 步，如果 PF+P_F^+PF+​ 不是全集 UUU，则依次将 LR 类属性跟 PPP 组合起来求闭包，只要其闭包是全集 UUU，就是候选码。

# 最小函数依赖集

如果函数依赖集 FFF 满足下列条件，则称 FFF 为一个最小函数依赖集，或称极小函数依赖集或最小覆盖。

  1. FFF 中的任一函数依赖的右部仅有一个属性，即无多余的属性；
  2. FFF 中不存在这样的函数依赖 X→AX→AX→A，使得 FFF 与 F−{X→A}F-\{X→A\}F−{X→A} 等价，即无多余的函数依赖；
  3. FFF 中不存在这样的函数依赖 X→AX→AX→A，XXX 有真子集 ZZZ 使得 FFF 与 F−{X→A}⋃{Z→A}F-\{X→A\}⋃\{Z→A\}F−{X→A}⋃{Z→A} 等价，即去掉各函数依赖左边的多余属性。 即：
    1. 所有函数依赖的右侧只有一个属性。
    2. 没有冗余的函数依赖。
    3. 所有函数依赖的左侧没有冗余的属性。

例：关系模式 R(U,F)R(U,F)R(U,F)，R(A,B,C,D,E),F={A→BC,A→E,A→D,D→E,AC→D}R(A,B,C,D,E), F=\{A→BC,A→E,A→D,D→E,AC→D\}R(A,B,C,D,E),F={A→BC,A→E,A→D,D→E,AC→D}

# 无损连接

分解及无损连接的定义：略，见教材 P288

无损连接是指将一个关系模式分解成若干个关系模式后，通过自然连接和投影等运算仍能还原到 原来的关系模式，则称这种分解为无损连接分解。

定理：关系模式 R(U,F)R(U,F)R(U,F) 的一个分解 ρ={R1(U1,F1),R2(U2,F2)}ρ =\{ R_1(U_1,F_1),R_2(U_2,F_2)\}ρ={R1​(U1​,F1​),R2​(U2​,F2​)}，具有无损连接的充分必要的条件是： U1⋂U2→U1－U2∈F+U_1⋂U_2→U_1－U_2∈F^+U1​⋂U2​→U1​－U2​∈F+ 或 U1⋂U2→U2－U1∈F+U_1⋂U_2→U_2－U_1∈F^+U1​⋂U2​→U2​－U1​∈F+。

注意：这个定理只适用于分解为两个子模式的情况，分解为多个子模式的时候不适用。

例：对给定的关系模式R(U,F)R(U,F)R(U,F)，U={A,B,C}U=\{A,B,C\}U={A,B,C}，F={A→B}F=\{A→B\}F={A→B}，有两个分解：ρ1={AB,BC}ρ1 =\{AB,BC\}ρ1={AB,BC} 和 ρ2={AB,AC}ρ_2 =\{AB,AC\}ρ2​={AB,AC}。请判断这两 个分解是否无损。

# 保持函数依赖

定义：设关系模式 R(U,F)R(U,F)R(U,F) 的一个分解 ρ=R1(U1,F1),R2(U2,F2),…,Rk(Uk,Fk)ρ ={R_1(U_1,F_1), R_2(U_2,F_2),…,R_k(U_k,F_k)}ρ=R1​(U1​,F1​),R2​(U2​,F2​),…,Rk​(Uk​,Fk​)，如果 F+=(⋃i=1kFi)+F^+=(\bigcup_{i=1}^{k} F_i)^+F+=(⋃i=1k​Fi​)+，则称分解 ρρρ 保持函数依赖。
